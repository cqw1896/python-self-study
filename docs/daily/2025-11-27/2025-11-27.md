**遇到的困难**： 

 彻底删除已经关联的远程仓库后，推送失败

```bash
git remote remove origin
```

**解决方案**：

```bash  
# 1. 确认当前分支是 main（大多数情况都是）
git branch
# 如果前面有 `* main` 就说明已经在 main 分支，直接继续； 如果不是，就切换：
git checkout main
# 或者直接把当前分支改名为 main（推荐）：
git branch -M main

# 2.重新添加正确的远程仓库地址
git remote add origin git@github.com:<username>/python-self-study.git
# 如果ssh远程仓库地址之前使用了https方式，需要先更改为ssh方式
# https方式：git remote add origin https://github.com/<username>/python-self-study.git
git remote set-url origin git@github.com:<username>/python-self-study.git

# 3. 执行第一次推送
git push -u origin main
# 避免第一次推送时加 -u
git config --global push.autoSetupRemote true
```
